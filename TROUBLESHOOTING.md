# Troubleshooting Guide

## 404 Errors for Next.js Internal Files

If you're seeing errors like:
- `Failed to load resource: app-pages-internals.js (404)`
- `Failed to load resource: layout.css (404)`

### Solution

1. **Stop the dev server** (Ctrl+C)

2. **Clear the build cache:**
   ```bash
   # Windows PowerShell
   Remove-Item -Recurse -Force .next
   
   # Or manually delete the .next folder
   ```

3. **Clear node_modules and reinstall (if needed):**
   ```bash
   Remove-Item -Recurse -Force node_modules
   npm install
   ```

4. **Restart the dev server:**
   ```bash
   npm run dev
   ```

5. **Clear browser cache:**
   - Press `Ctrl+Shift+R` (Windows) or `Cmd+Shift+R` (Mac) for hard refresh
   - Or clear browser cache manually

### Why This Happens

These files (`app-pages-internals.js`, `layout.css`) are automatically generated by Next.js during the build process. If the `.next` directory is corrupted or missing, Next.js can't serve these files, causing 404 errors.

### Additional Checks

1. **Verify Next.js is installed correctly:**
   ```bash
   npm list next
   ```

2. **Check for port conflicts:**
   - Next.js defaults to port 3000
   - Make sure no other process is using port 3000
   - Or specify a different port: `npm run dev -- -p 3001`

3. **Verify environment variables:**
   - Make sure `.env.local` exists with required variables
   - Check that `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` are set

4. **Check console for errors:**
   - Look for any error messages in the terminal where `npm run dev` is running
   - Check browser console for additional error messages

### If Issues Persist

1. **Delete all build artifacts:**
   ```bash
   Remove-Item -Recurse -Force .next
   Remove-Item -Recurse -Force node_modules
   npm install
   npm run dev
   ```

2. **Verify file structure:**
   - Make sure `app/layout.tsx` exists
   - Make sure `app/globals.css` exists
   - Check that all imports are correct

3. **Check Next.js version:**
   ```bash
   npm list next
   ```
   - Should be version 14.2.5 or compatible
   - Update if needed: `npm install next@latest`

## Common Issues

### Authentication Errors
- **Issue:** "Authentication required" errors
- **Solution:** Make sure you're signed in and Supabase is configured correctly

### Database Errors
- **Issue:** Projects not loading
- **Solution:** Verify Supabase connection and RLS policies are set up correctly

### Build Errors
- **Issue:** TypeScript or ESLint errors
- **Solution:** Fix linting errors or temporarily disable strict mode

### Port Already in Use
- **Issue:** "Port 3000 is already in use"
- **Solution:** 
  ```bash
   # Find and kill the process using port 3000
   netstat -ano | findstr :3000
   taskkill /PID <PID> /F
   
   # Or use a different port
   npm run dev -- -p 3001
   ```

## Getting Help

If you continue to experience issues:
1. Check the browser console for detailed error messages
2. Check the terminal where `npm run dev` is running
3. Verify all environment variables are set correctly
4. Make sure Supabase is configured and running
5. Check that the database migration has been run
